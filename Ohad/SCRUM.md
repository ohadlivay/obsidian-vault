# Roles
* PRODUCT OWNER: interacts with customer, understands key system needs and writes user stories. 
* TEAM: 2-9 members. 
* SCRUM MASTER: assigns tickets to team member. orchestrates the sprint. 
# Product Backlog
a priority queue of user-stories we need per project, spanning over many sprints.
# Sprint Backlog
like Product Backlog but with the scope of a single sprint and where each user story is turned into many tickets, which the scrum master then assigns to team members.
# Task board
a collection of tickets and their origin user story and their current status {pending, in process, to verify, done}

Daily Scrum - daily synchronization
